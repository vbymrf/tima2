# Редактор контента (посты, статьи, медиа, истории)

> Единый редактор публикаций (фич-модуль `feature-editor`, [module-boundaries.md](../02-architecture/module-boundaries.md) §6). Формат: **plain text + entities** (§5 там же). API: `POST /posts`, `/drafts`. Не путать со строкой ввода сообщений в чатах (ЗОНА 4, [03-personal-chat](./03-personal-chat.md)).

## Режимы

| Режим | Откуда открывается | kind | Особенности |
|-------|--------------------|------|-------------|
| Пост канала | [06-channel](./06-channel.md) [+ Пост] | text | Короткий текст + медиа-карусель |
| Статья | Канал → [+ Пост] → «Статья» | article | Заголовок, форматирование, медиа в тексте |
| Медиа-пост | Окно 3 [+], окно 5 | photo/video | Медиа первично, подпись + атрибуты |
| История | Окно 4 [+ История] | — | Медиа + TTL 24ч + аудитория ([11-stories](./11-stories.md)) |

От чьего имени: пользователь / канал (роль owner/admin/author) / виртуальный пользователь (`acting_as`, [virtual-users.md](../01-product/virtual-users.md)) — выбор при открытии, если контекстов несколько.

[**B**] [*I*] [U] [~~S~~] [`код`] [❝ цитата] [H заголовок] [• список] [🔗 ссылка] [@ упоминание] [# атрибут]
:::

::: row
[📷 Медиа] [🖼 В текст] [👁 Предпросмотр] [🕓 Отложить]
:::

> Панель форматирования применяет **entities** к выделению (bold, italic, underline, strikethrough, code, quote, heading, list_item, link, mention, hashtag).
> **[# атрибут]** — атрибуты поста (= хэштеги, единый реестр): автодополнение; нет совпадений → создание нового с предложением жанра ([35-attributes-genres](./35-attributes-genres.md)). `#хэштег`, набранный прямо в тексте, — entity `hashtag`, резолвится в тот же реестр.
> **[📷 Медиа]** — карусель вложений поста (до 10, [19-attachments-media](./19-attachments-media.md)); **[🖼 В текст]** (только статья) — entity `media` в позицию курсора.
> **[🕓 Отложить]** → выбор даты/времени → `scheduled_at` (пост со статусом scheduled, виден автору в списке «Отложенные»).

> Выбор качества: сжато / оригинал / авто. Публикация → пост `kind photo|video` в медиа-ленту; фоновая загрузка с очередью.

## Черновики

- Автосохранение каждые 5 с локально + `PUT /drafts` (debounce 30 с).
- Закрытие редактора с непустым контентом → «Сохранить черновик?» [Сохранить]* [Удалить].
- Список черновиков — в точке входа режима (канал → «Черновики», окно 5).
- Черновик привязан к автору-контексту (канал / ВП / пользователь).

## Предпросмотр и публикация

::: modal
## Предпросмотр

(рендер поста как в ленте: entities, карусель, атрибуты)

[Опубликовать]* [← Редактировать]
:::

> Канал с премодерацией: автор (не admin) публикует → статус **pending**, баннер «Отправлено на модерацию»; admin видит очередь → [Одобрить]/[Отклонить] (`/posts/{id}/approve|reject`).

## Ошибки и состояния

> **Нет сети:** публикация встаёт в offline-очередь ([sync-offline.md](../04-data/sync-offline.md) §3), черновик сохранён локально.
> **Медиа не загрузилось:** пост не публикуется, ошибка на карточке вложения [Повторить].
> **Права отозваны** (роль author снята во время редактирования): предупреждение, контент остаётся черновиком.

## Права

| Кто | Что может |
|-----|-----------|
| owner/admin канала | Посты и статьи, публикация мгновенно, модерация очереди |
| author канала | Посты и статьи; при премодерации → pending |
| Пользователь | Медиа-посты, истории, посты на свою страницу |
| Виртуальный пользователь | То же от имени ВП (`acting_as`; действует владелец/оператор, [29](./29-blogger-media-window.md)/[30](./30-blogger-news-window.md)) |
| Временный аккаунт | Только личный контент; публикации в каналы — после апгрейда |

## Адаптация

> **Телефон:** полноэкранный редактор. **Планшет/десктоп:** модал ~700px; статья — двухпанельный режим (редактор | живой предпросмотр).
