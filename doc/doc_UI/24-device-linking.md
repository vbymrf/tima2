# Привязка Windows через телефон (QR)

> **Модель доверия:** телефон — «ядро доверия». Windows генерирует ключи; мобильное приложение (с App Attest / Play Integrity) подтверждает.

## Шаг 1 — Windows-клиент (первый запуск)

::: card

### Вход на этом устройстве

Сгенерирована пара ключей. Отсканируйте QR телефоном с установленным приложением.

┌─────────────────┐
│   [ QR CODE ]   │
└─────────────────┘

Код: `ABCD-1234`

[Обновить QR]

:::

## Шаг 2 — Мобильное приложение

::: modal
## Подтвердить вход Windows?

Устройство: Windows PC
Публичный ключ: `…`

[Доверить]* [Отклонить]{danger}

:::

> Пользователь уже авторизован; attestation пройдена на телефоне.

> После привязки Windows получает **собственные wrapped keys** (обёрнутые на его identity-ключ) для sync защищённой истории — backfill 90 дней. Per-device ключи — см. [key-lifecycle.md](../03-security/key-lifecycle.md), [23-auth-recovery.md](./23-auth-recovery.md).

## Шаг 3 — Успех

::: alert success
Windows-клиент привязан. Сессия помечена как «доверенная через мобильное устройство».
:::

## Список сессий

::: card

| Сессия | Статус |
|--------|--------|
| Android (основной) | Доверенное |
| Windows PC | Доверенное через телефон |
| Web | Ожидает подтверждения |

[Отозвать]{danger}

:::

## Ошибки

::: alert error
QR истёк. [Обновить]*
:::

::: alert warning
Attestation не пройдена. Вход только с мобильного устройства.

:::
