**Карточка обращения показывает:** identity (какой ВП / основной аккаунт), источник, отправителя, preview, статус, ответственного, действия.

| Действие | Смысл |
|----------|-------|
| Открыть | Переход в **настоящий** чат обращения — экран личного чата с бейджем ВП и панелью оператора |
| Взять | assignee = я (виден всей команде) |
| Отложить | snooze до времени; карточка вернётся |
| Закрыть | Обращение решено; из «Входящих» уходит |

> **Статус и ответственный — серверные**, общие для владельца и операторов ВП. Статусы: `new → taken → snoozed → closed`.
> События основного аккаунта (реакции, упоминания) — личные: у них только прочитано/скрыть, «Взять» не показывается.

> Настройки (`social_inbox_preferences`) **синхронизируются между устройствами** ([sync-offline.md](../04-data/sync-offline.md)); read-state и assignment — серверные. Push и тихие часы исполняет общий модуль уведомлений ([26-notifications](./26-notifications.md)) — правила маршрутизации только решают, куда попадает карточка и её приоритет.

## Состояния

> **Empty («Входящие»):** «Нет новых обращений» + [Настроить правила].
> **Нет ВП:** вкладка «Входящие» показывает только события основного аккаунта; подсказка «Создайте виртуального пользователя» → настройки ([25-settings-help-bugs.md](./25-settings-help-bugs.md)); создание ВП не требует блогерского режима (окно 6 — лишь один из UI для управления ими).
