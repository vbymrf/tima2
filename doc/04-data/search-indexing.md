# Поиск и индексация

> Каноника: приватное ищется только локально, публичное — на сервере ([ADR-0007](../adr/0007-search-split.md)). UI: [17-global-search](../doc_UI/17-global-search.md).

## 1. Два контура

| Контур | Данные | Движок | Где |
|--------|--------|--------|-----|
| Локальный | Личные чаты, личные группы, личные коллекции, избранное | SQLite **FTS5** (SQLDelight) | Устройство |
| Серверный | Публичные посты/сообщения, каналы, профили, каталог, публичные коллекции, **атрибуты и жанры** | PostgreSQL FTS → OpenSearch (по триггеру) | Сервер |

## 2. Локальный индекс (приватное)

- Индексируется расшифрованный текст из локального кэша при записи сообщения в SQLDelight: `INSERT INTO message_fts(message_id, chat_id, text)`.
- Токенизатор: `unicode61` + `remove_diacritics`; префиксный поиск (`prefix='2 3'`) для «поиска на лету».
- Вложения: индексируются имя файла и подпись; содержимое документов не парсится (MVP).
- Охват = данные на устройстве (по умолчанию 90 дней + всё скачанное). Пустой результат по старой истории — честно показываем «искать можно только в загруженной истории».
- Индекс лежит в той же шифрованной локальной БД (ключ — производный от пароля, [key-lifecycle.md](../03-security/key-lifecycle.md) §1).

## 3. Серверный поиск (публичное)

**MVP — PostgreSQL FTS:**
- `posts.fts` (GIN, генерируемая колонка) — уже в схеме; аналогично для публичных `group_messages` (materialized tsvector по партициям) и профилей (`username`, `display_name`, trigram `pg_trgm` для опечаток).
- Русская + английская конфигурация (`russian`, `simple` для тегов/username).

**Миграция на OpenSearch** — по триггеру (p95 поиска > 500 мс или > 10M документов):
- Индексы: `posts`, `public_messages`, `profiles`, `catalog`; синхронизация через outbox-таблицу + воркер.
- API-фасад `search` в бэкенде не меняется — клиент миграции не видит.

## 4. Глобальная строка поиска (клиент)

```
Запрос пользователя
   ├─► локальный FTS5 (приватное)      ─┐
   └─► GET /search?q=&type=            ─┴─► объединение и ранжирование на клиенте
Фильтры-табы: Люди · Чаты · Сообщения · Медиа · Коллекции
```

- Результаты помечаются источником; приватные всегда выше при равном скоре.
- Debounce 300 мс; серверный запрос отменяется при вводе.
- Поиск внутри конкретного чата: приватный чат — только FTS5 с фильтром `chat_id`; публичный — серверный с фильтром.

## 5. Что не ищется никогда

- Содержимое защищённых чатов на сервере (физически невозможно — ciphertext).
- Контент заблокировавших пользователя авторов (фильтр blocklist на выдаче).
- Soft-deleted контент.
