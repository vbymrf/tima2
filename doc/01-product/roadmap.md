# Этапы создания (roadmap)

> Каноничный план разработки. Команда: 2–3 человека. Исследовательские планы («2 недели», «3 месяца» из docx) заменены этим документом.
> Принцип: **форматы данных и крипто-слои правильные с первого дня** — миграций «прикрутим шифрование потом» не будет.

## Обзор фаз

| Фаза | Содержание | Результат |
|------|-----------|-----------|
| 0. Фундамент | Инфраструктура, скелеты, крипто-SDK | Пустое приложение ходит на сервер по TLS |
| 1. Личное общение | Чаты 1:1 с шифрованием, медиа, устройства | Защищённый мессенджер 1:1 |
| 2. Звонки | LiveKit, аудио/видео 1:1 и группы | Рабочие звонки |
| 3. Группы и каналы | Личные группы (GK), публичные группы, каналы | Окно 2 функционально |
| 4. Ленты и социальное | Ленты, реакции, истории, коллекции, окна 3–5 | Полный продукт-контур |
| 5. Полировка и бета | Ratchet (PFS), поиск, уведомления, аудит | Закрытая бета |
| 6. Публичный релиз | Production HSM escrow, аудит крипто, магазины | Релиз |

Фазы 1–3 — последовательные; внутри фаз 4–5 задачи параллелятся.

---

## Фаза 0 — Фундамент

**Backend (Go):**
- Репозиторий, CI, модульный монолит: `auth`, `users`, `messages`, `media`, `keys` как внутренние пакеты.
- **Schema-first с первого дня**: каталог `schema/` (OpenAPI + Protobuf) + кодогенерация Go-типов и KMP-клиента ([ADR-0009](../adr/0009-schema-first-api.md)).
- PostgreSQL + миграции (schema из [data-model.md](../02-architecture/data-model.md)), Redis, MinIO — docker-compose ([server-setup.md](../07-deployment/server-setup.md)).
- Caddy: TLS, маршрутизация REST + WS.
- Регистрация по SMS + обязательный email + резервные коды; **временный режим** (гостевой аккаунт без телефона/email, [key-lifecycle.md](../03-security/key-lifecycle.md) §7); JWT-сессии, device registry.

**Клиент (KMP + Compose):**
- Проект из KMP Wizard: Android + iOS + Desktop, единый Compose UI.
- Оболочка приложения: 5 окон, переключатель, навигация ([01-app-shell](../doc_UI/01-app-shell.md)).
- SQLDelight, Ktor Client, Koin; Konsist-правила в CI.

**Крипто-SDK (`messenger-crypto`, KMP-модуль поверх Kodium):**
- `EnvelopeCipher`, `WrappedKeyService`, `EscrowModule` (пока stub-ключ), `MessageSerializer` (Protobuf + zstd).
- Генерация identity keys, хранение в Keystore/Secure Enclave.
- Тест-векторы форматов из [crypto-protocol.md](../03-security/crypto-protocol.md).

**Выход фазы:** регистрация, вход, пустые окна, ключи устройств на сервере.

## Фаза 1 — Личное общение (1:1)

- Отправка/приём текста через WS: конверт + wrapped keys + escrow blob (stub) с первого сообщения.
- Экран личного чата ([03-personal-chat](../doc_UI/03-personal-chat.md)): статусы ✓/✓✓, «печатает», ответ с цитатой.
- Вложения: presigned upload в MinIO, клиентское шифрование, фото + файлы ([19-attachments-media](../doc_UI/19-attachments-media.md)).
- Голосовые сообщения: запись → Opus → Kodium → MinIO (**не** LiveKit).
- Мультиустройство: привязка Windows по QR через телефон ([24-device-linking](../doc_UI/24-device-linking.md)), per-device wrapped keys.
- Аттестация: App Attest (iOS), Play Integrity (Android) ([client-attestation.md](../03-security/client-attestation.md)).
- Телефонная книга, список чатов ([02-phone-home](../doc_UI/02-phone-home.md)).

**Выход фазы:** защищённый текстово-медийный мессенджер 1:1 на 3 платформах.

## Фаза 2 — Звонки

- Деплой LiveKit (self-hosted) + TURN.
- Аудио/видео 1:1 (P2P), групповые до 20 (SFU), демонстрация экрана.
- Окно 0, все состояния звонка ([21-call](../doc_UI/21-call.md)), история звонков.
- Записи звонков **нет** ([ADR-0006](../adr/0006-livekit-media-policy.md)).

**Выход фазы:** звонки Android ↔ iOS ↔ Desktop.

## Фаза 3 — Группы, каналы, сообщества

- Личные группы: `GroupKeyManager` — GK, ротация (N сообщений / вход / выход), wrapped_GK per участник.
- Публичные группы: plaintext-хранение, ветки (threads), премодерация, slow mode.
- Каналы (**только посты**): роли owner/admin/author, посты через редактор ([34-content-editor](../doc_UI/34-content-editor.md)) — текст + entities, черновики; комментарии (общая подсистема). Границы модулей — [module-boundaries.md](../02-architecture/module-boundaries.md).
- **Сообщества**: контейнер групп/каналов/аудио-чатов, подписка на сообщество, уровни доступа preview/open/restricted ([communities.md](./communities.md)).
- Мастер создания ([33-create-group-channel](../doc_UI/33-create-group-channel.md)).
- Аудио-чаты — только внутри сообществ (авто-создание сообщества при «отдельном» создании), опциональная привязка к группе/каналу ([07-voice-chat](../doc_UI/07-voice-chat.md)).
- Инвайты и QR — в элемент или сообщество ([28-qr-invites](../doc_UI/28-qr-invites.md)), базовая модерация и жалобы.
- Папки каталога (личная сортировка).
- Лимит MVP: группы до 100 участников (цель — 1000+, см. NFR).

**Выход фазы:** окно 2 (Новостная лента: чаты, сообщества, каталог) работает.

## Фаза 4 — Ленты и социальное

- Общая лента: скоринг по оценкам + атрибуты/жанры (реестр, одобрение «автор + репутация»); лента друзей: fan-out публичных полок ([feed-ranking.md](../04-data/feed-ranking.md)); [+]/[−] для публичного.
- Избранное: две полки; личная — шифрованная, доступ по запросу (wrapped shelf_key).
- Шкала 9 эмоций + раздельные рейтинговые счётчики «+/−» по пользователям и группам ([18-content-actions](../doc_UI/18-content-actions.md)).
- Медиа-лента: режимы «Лента»/«Слайды» (окно 3), публичные коллекции; медиа-посты и статьи через тот же редактор (отложенная публикация).
- Окно 4 «Социальное взаимодействие»: агрегатор карточек (Входящие/Мои треды/Реакции/Коллекции), истории (TTL 24ч), правила маршрутизации, серверные статусы обращений ([10-social-interaction](../doc_UI/10-social-interaction.md)). Карточки `entity_message` в MVP шлют владельцы/админы сущностей (`/inbox/notify`) и система; боты — после Bot API.
- **Виртуальные пользователи**: создание, операторы, managed inbox, ключи без устройств, передача ([virtual-users.md](./virtual-users.md)).
- Окно 5: профиль, избранное (сквозное), подписки.
- Комментарии ([15-comments](../doc_UI/15-comments.md)), пересылка/репост ([20-share-repost](../doc_UI/20-share-repost.md)).

**Выход фазы:** все 5 окон функциональны.

## Фаза 5 — Полировка и закрытая бета

- **Double Ratchet (путь A)** поверх конвертов — PFS для активных сессий 1:1.
- Поиск: локальный FTS5 (приватное) + PG FTS (публичное) ([ADR-0007](../adr/0007-search-split.md)).
- Уведомления: 3 уровня, группировка, тихие часы; push (FCM/APNs).
- Настройки, помощь, баг-репорты; блокировка приложения, чёрный список, «невидимка», временные чаты.
- Наблюдаемость: метрики, логи, алерты; нагрузочное тестирование.
- Блогерские окна 6–7 — минимальная версия (статистика, виртуальные пользователи как псевдонимы).

**Выход фазы:** закрытая бета (TestFlight / internal track / MSIX).

## Фаза 6 — Публичный релиз (readiness gate)

Обязательные условия ([ADR-0005](../adr/0005-kodium-readiness-gate.md)):

- [ ] Production escrow: HSM/Nitro Enclave, M-of-N, audit log ([escrow-legal-access.md](../03-security/escrow-legal-access.md)).
- [ ] Независимый аудит Kodium + `messenger-crypto`.
- [ ] Верификация Signed PreKey в X3DH; фаззинг десериализации.
- [ ] Политики: конфиденциальность, формулировки «защищённый чат» в UI.
- [ ] Резервное копирование и DR-процедуры ([server-setup.md](../07-deployment/server-setup.md#резервное-копирование)).
- [ ] Публикация в App Store / Google Play / магазины Windows.

**Bot API** ([bot-api.md](../05-api/bot-api.md): RPC-контур, webhook/polling, scopes, SDK; aiogram-подобный фреймворк), стикеры/GIF, распознавание голосовых, отложенная отправка, напоминания, экспорт чатов, LiveKit E2EE для звонков, Egress для публичных комнат (новый ADR), OpenSearch, рекламный кабинет блогера, маркетплейс передачи ВП, A/B-инфраструктура.
