# Сообщества

> **Сообщество** — административный контейнер, объединяющий группы (публичные и личные), каналы и аудио-чаты. Каноническая модель домена groups/channels/communities. UI: [32-community](../doc_UI/32-community.md), создание — [33-create-group-channel](../doc_UI/33-create-group-channel.md). Схема — [data-model.md](../02-architecture/data-model.md) §3.

## 1. Два вида объединений — не путать

| | **Папка** | **Сообщество** |
|---|-----------|----------------|
| Кто создаёт | Пользователь для себя | Администратор для всех |
| Где живёт | Только на стороне пользователя (организация его каталога) | Серверный объект с собственной страницей |
| Что содержит | Ссылки на любые добавленные чаты/каналы/сообщества | Группы, каналы, аудио-чаты, принадлежащие сообществу |
| Видимость | Никому, кроме владельца | Публичная страница (состав — по правам доступа) |
| Подписка | Нет такого понятия | **Единица подписки** |
| Модель данных | `chat_folders` / `chat_folder_items` | `communities` + `community_id` у groups/channels/voice_rooms |

Папка может содержать сообщество; сообщество не может содержать папку.

## 2. Структура сообщества

```
Сообщество «DevTeam»
├── Канал «Анонсы»            (open — контент для подписчиков)
├── Публичная группа «Общий»   (preview — виден и без подписки)
├── Личная группа «Ядро»       (restricted — только по приглашению, E2E)
└── Аудио-чат «Standup»        (open)
```

- Элемент принадлежит **не более чем одному** сообществу (`community_id`).
- Группы и каналы могут существовать **вне** сообщества (standalone).
- **Аудио-чат существует только внутри сообщества** — без сообщества не создаётся (`community_id` обязателен). Пути создания:
  - со страницы сообщества → создаётся в нём;
  - «отдельный» аудио-чат из мастера → **автоматически создаётся сообщество-обёртка** (имя = имя аудио-чата);
  - из группы/канала, входящих в сообщество → аудио-чат создаётся в том же сообществе и прикрепляется к группе/каналу (`attached_type/attached_id` — слабая ссылка отображения);
  - из standalone-группы/канала → мастер сначала создаёт сообщество, включающее и группу/канал, и новый аудио-чат.
- Вложенность сообществ в сообщества запрещена.

## 3. Подписка и доступ

**Подписка оформляется на сообщество** (одно действие — весь контейнер). Доступ к каждому элементу задаётся администратором отдельно (`community_access`):

| Уровень | Без подписки | С подпиской | Пример |
|---------|--------------|-------------|--------|
| `preview` | ✅ читает | ✅ читает | Витрина: публичная группа или канал, показываемые всем |
| `open` | ❌ (виден в списке, контент закрыт) | ✅ читает, контент в лентах | Основные каналы/группы сообщества |
| `restricted` | ❌ не виден или виден без доступа* | ❌ только по приглашению/роли | Личная группа ядра команды (E2E — членство через wrapped keys) |

\* видимость restricted-элемента в списке настраивается («показывать название» / «скрыть полностью»).

Правила:

1. Контент `open`-элементов попадает в ленты подписчиков сообщества (fan-out по `subscriptions(target_type='community')`).
2. `preview`-элементы — «то одно, что можно показывать без подписки»: их контент доступен любому, служит витриной сообщества.
3. `restricted` + личная группа: подписка на сообщество **не даёт** членства — оно оформляется приглашением, с выдачей wrapped_GK ([crypto-protocol.md](../03-security/crypto-protocol.md) §4). Шифрование и матрица безопасности элементов не меняются от вхождения в сообщество ([content-security-matrix.md](./content-security-matrix.md)).
4. Подписка на standalone-группу/канал (вне сообществ) работает как раньше — напрямую.
5. При добавлении существующей группы/канала в сообщество прямые подписчики элемента сохраняются; новые подписки идут через сообщество.

## 4. Роли и модерация

Роли **сообщества** (подсистема `memberships`, target_type='community'):

| Роль | Права |
|------|-------|
| owner | Всё: состав элементов, роли, удаление сообщества |
| admin | Добавление/удаление элементов, настройка `community_access`, назначение модераторов, инвайты |
| moderator | Модерация контента во всех `open`/`preview` элементах: удаление сообщений, блокировка участников |

Роли **элемента** (подсистема `memberships`: группа — owner/admin/moderator/member; канал — owner/admin/author; аудио-чат — owner/speaker) продолжают действовать внутри элемента. Приоритет: роль сообщества даёт права во всех его элементах; локальная роль — только в своём. Конфликтов нет: права складываются (действует максимум).

Модерация: жалобы (`reports`) на контент элемента видят и модераторы элемента, и модераторы сообщества.

## 5. Создание (wizard)

Единый мастер создания ([33-create-group-channel](../doc_UI/33-create-group-channel.md)):

```
Шаг 1. Тип: Личная группа | Публичная группа | Канал | Аудио-чат | Сообщество
Шаг 2. Имя, описание, аватар
Шаг 3. Размещение: standalone / в моё сообщество (для аудио-чата standalone → авто-сообщество)
Шаг 4. Доступ: community_access (если в сообществе), кто может писать (канал: admins|admins+authors),
        инвайт-политика, модерация (премодерация постов вкл/выкл)
Шаг 5. (личная группа) — первые участники → первая ротация GK
```

Создатель получает роль owner элемента; если создано авто-сообщество — owner сообщества.

## 6. Каталог и папки

- Каталог (окно 2/3, вкладка «Каталог») показывает **добавленное пользователем**: сообщества, standalone-группы/каналы, аудио-чаты.
- Папки — личная сортировка этого списка: `chat_folders` (имя, иконка, позиция) + `chat_folder_items` (что лежит; `target_type` включает `community`). Логика полностью клиент-серверная CRUD без прав/шаринга: папку никто, кроме владельца, не видит.
- Один элемент может лежать в нескольких папках владельца.

## 7. Жизненный цикл

| Событие | Поведение |
|---------|-----------|
| Удаление сообщества | Группы/каналы становятся standalone (не удаляются); аудио-чаты удаляются (см. ниже); подписчики сообщества теряют подписку (уведомление) |
| Удаление элемента из сообщества | Элемент → standalone; его прямые участники/подписчики сохраняются |
| Передача владения | owner → другому админу (сообщества и элемента независимо) |
| Авто-сообщество аудио-чата | Живёт как обычное; если удалить аудио-чат — пустое сообщество остаётся (можно удалить отдельно) |
| Удаление сообщества с аудио-чатами | Аудио-чаты не могут стать standalone — удаляются вместе с сообществом (с подтверждением); группы/каналы становятся standalone |
