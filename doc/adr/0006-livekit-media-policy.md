# ADR-0006: Медиа-политика LiveKit — звонки вне app-E2E, запись private запрещена

**Статус:** принят · **Дата:** 2026-07-12

## Контекст

Живые звонки идут через LiveKit (WebRTC/SRTP). SFU по умолчанию видит медиапоток. LiveKit Egress (запись) декодирует поток в plaintext на сервере — это несовместимо с политикой клиентского шифрования. В исследованиях запись звонков через Track Egress планировалась как фича MVP.

## Решение

1. **Звонки (1:1 и групповые)** — LiveKit: P2P только для 1:1, группы — через SFU. Защита — SRTP + TLS 1.3 signaling. Это **не** app-level E2E, и UI не заявляет обратного ([27-security-privacy.md](../doc_UI/27-security-privacy.md)).
2. **Запись звонков выключена. LiveKit Egress для private-комнат не разворачивается и не конфигурируется.**
3. **Голосовые сообщения** (async) — не через LiveKit: запись на клиенте → Opus → шифрование Kodium → MinIO. Полноценное клиентское шифрование.
4. Возможное будущее: Egress только для **публичных** комнат с постоянным баннером «ведётся запись» и отдельной политикой — потребует нового ADR; политика: [recording-policy.md](../06-realtime/recording-policy.md).
5. LiveKit E2EE (insertable streams) — опция для приватных звонков в будущем, ортогональна Kodium.

## Последствия

- Из roadmap исключена задача «запись звонков через Egress» (была в исследованиях).
- Хранение голосовых — единый путь с остальными вложениями ([media-storage.md](../04-data/media-storage.md)).
