# ADR-0007: Раздельный поиск — приватное локально, публичное на сервере

**Статус:** принят · **Дата:** 2026-07-12

## Контекст

Приватный контент хранится на сервере только как ciphertext — сервер физически не может его индексировать. В исследованиях фигурировали Elasticsearch и OpenSearch одновременно.

## Решение

| Контент | Индекс | Где |
|---------|--------|-----|
| Личные чаты, личные группы, личные коллекции | **SQLite FTS5** | Только на устройстве, по расшифрованному локальному кэшу |
| Публичные группы, каналы, публичные медиа, профили, каталог | Серверный поиск | MVP: **PostgreSQL FTS**; при росте: **OpenSearch** |

- Глобальная строка поиска ([17-global-search.md](../doc_UI/17-global-search.md)) объединяет оба источника на клиенте.
- OpenSearch (не Elasticsearch): открытая лицензия, совместимый API. Триггер миграции — деградация PG FTS под нагрузкой ([scaling.md](../07-deployment/scaling.md)).

## Последствия

- Поиск по приватной истории работает только по данным, синхронизированным на устройство ([sync-offline.md](../04-data/sync-offline.md)).
- Детали схем индексации: [search-indexing.md](../04-data/search-indexing.md).
