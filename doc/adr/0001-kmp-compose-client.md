# ADR-0001: Клиент на Kotlin Multiplatform + Compose Multiplatform

**Статус:** принят · **Дата:** 2026-07-12

## Контекст

Нужен один клиент под Android, iOS и Desktop (Windows) силами команды 2–3 человека. В исследованиях рассматривалась альтернатива: React Native + JSON UI + Native Modules поверх KMP-ядра (для кастомизации дизайна чатов).

## Решение

- **KMP** — общая бизнес-логика, сеть, крипто (Kodium — pure Kotlin), локальная БД.
- **Compose Multiplatform** — единый UI для всех платформ. Все 32 UI-спецификации ([doc_UI](../doc_UI/00-index.md)) написаны в формате WireMD под Compose.
- Платформенный код (expect/actual): микрофон, камера, push-уведомления, Keystore/Secure Enclave, аттестация.
- Библиотеки: **SQLDelight** (локальная БД), **Ktor Client** (HTTP/WS), **Koin** (DI), **Konsist** (архитектурные правила в CI).

## Отклонено

- **React Native-гибрид** — двойной UI-стек непосилен для маленькой команды; кастомизация дизайна чатов (блогерская фича) решается темизацией Compose + JSON-описанием темы.
- Полностью нативные клиенты (Swift + Kotlin + C#) — троекратная стоимость.

## Последствия

- Версии стека фиксируются в [tech-stack.md](../02-architecture/tech-stack.md) и проверяются на актуальность при старте разработки (не берутся из старых исследований).
- Микрофон/камера/уведомления пишутся дважды (Android/iOS) — это ожидаемая цена KMP.
- **Web-клиент — в планах** (пост-MVP): Compose Multiplatform for Web/Wasm, доверие через QR-привязку телефоном; решение о старте — отдельным ADR после оценки зрелости Wasm-таргета.
