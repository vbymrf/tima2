# Политика записи

> Каноника: [ADR-0006](../adr/0006-livekit-media-policy.md). Причина: LiveKit Egress декодирует поток в plaintext на сервере — несовместимо с политикой защищённого контента.

## Правила

1. **Запись звонков и голосовых чатов отключена.** LiveKit Egress не разворачивается, не конфигурируется и отсутствует в docker-compose.
2. **Голосовые сообщения — не запись звонка.** Они записываются на устройстве, шифруются Kodium и загружаются в MinIO как обычное вложение ([crypto-protocol.md](../03-security/crypto-protocol.md) §5). Это единственный канонический способ «голоса в хранилище».
3. Клиентская запись звонка средствами ОС нам технически не подконтрольна; UI не предоставляет собственной кнопки записи.

## Если запись потребуется в будущем

Любой вариант — **новый ADR** + обновление этой политики. Оценённые варианты:

| Вариант | Совместимость с политикой | Условия |
|---------|--------------------------|---------|
| Запись на клиенте → Kodium → MinIO | ✅ | Согласие всех участников, индикатор в UI |
| LiveKit E2EE в комнате + Egress с ключами у клиентов | ✅ | Высокая сложность |
| Egress для **публичных** комнат | ⚠️ допустимо | Постоянный баннер «ведётся запись», публичная комната по определению не защищённая |
| Egress для приватных звонков | ❌ запрещено | — |

## Связанные обязательства

- В пользовательском соглашении фиксируется: сервис не записывает звонки.
- Любая телеметрия звонков ограничивается метаданными качества (битрейт, потери пакетов) — без медиасодержимого.
