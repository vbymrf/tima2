# Установка тулчейна для проекта TIMA

> Открой этот файл в редакторе (VS Code / Блокнот), а не в терминале — так удобнее.
> Команды PowerShell запускай **от имени администратора** (Пуск → «PowerShell» → ПКМ → «Запуск от имени администратора»).
> Копируй по одному блоку. После каждой установки **закрывай и открывай терминал заново** — иначе новые команды не видны.

---

## Что и зачем

| Компонент | Зачем | Обязателен сейчас? |
|-----------|-------|--------------------|
| **JDK 17** | Компиляция Kotlin, запуск крипто-тестов | **ДА** — минимум для старта |
| Docker Desktop | Сервер: PostgreSQL, Redis, MinIO, LiveKit | Позже (шаг сервера) |
| Android SDK | Сборка APK | Позже (шаг Android) |
| Эмулятор | Запуск APK без телефона | Позже |

**Минимум, чтобы я начал писать и проверять крипто-ядро, — только JDK 17.** Остальное можно поставить, когда дойдём до сервера и Android.

---

## Шаг 1. JDK 17 (обязательно)

```powershell
winget install --id EclipseAdoptium.Temurin.17.JDK -e
```

Закрой и открой терминал. Проверка (должно показать `17.x`):

```powershell
java -version
```

Если `java` не найдена после переустановки терминала — перезагрузи компьютер.

---

## Шаг 2. Docker Desktop (позже)

```powershell
winget install --id Docker.DockerDesktop -e
```

После установки:
1. Перезагрузи Windows.
2. Запусти «Docker Desktop» вручную (значок на рабочем столе), дождись статуса «Running».

Проверка:

```powershell
docker --version
```

## Шаг 3. Android SDK (позже)

### 3.1. Создать каталог и скачать инструменты

```powershell
$sdk = "$env:LOCALAPPDATA\Android\Sdk"
New-Item -ItemType Directory -Force -Path "$sdk\cmdline-tools" | Out-Null
Invoke-WebRequest -Uri "https://dl.google.com/android/repository/commandlinetools-win-11076708_latest.zip" -OutFile "$env:TEMP\cmdtools.zip"
Expand-Archive "$env:TEMP\cmdtools.zip" -DestinationPath "$sdk\cmdline-tools" -Force
Rename-Item "$sdk\cmdline-tools\cmdline-tools" "latest"
```

### 3.2. Прописать переменные окружения (постоянно)

```powershell
$sdk = "$env:LOCALAPPDATA\Android\Sdk"
[Environment]::SetEnvironmentVariable("ANDROID_HOME", $sdk, "User")
$p = [Environment]::GetEnvironmentVariable("Path","User")
[Environment]::SetEnvironmentVariable("Path", "$p;$sdk\cmdline-tools\latest\bin;$sdk\platform-tools;$sdk\emulator", "User")
```

**Закрой и открой терминал.**

### 3.3. Поставить компоненты

```powershell
sdkmanager "platform-tools" "platforms;android-34" "build-tools;34.0.0" "emulator" "system-images;android-34;google_apis;x86_64"
```

Принять лицензии (несколько раз ввести `y`):

```powershell
sdkmanager --licenses
```

Проверка:

```powershell
adb --version
```

## Шаг 4. Эмулятор (позже, если нет телефона под рукой)

```powershell
avdmanager create avd -n tima_test -k "system-images;android-34;google_apis;x86_64" -d pixel_6
```

Запуск (когда понадобится):

```powershell
emulator -avd tima_test
```

> Эмулятор НЕ заменяет реальный телефон для видеозвонков (нет настоящей камеры и сети). Для звонков нужны два реальных Android-устройства.

## Если что-то не работает

| Проблема | Решение |
|----------|---------|
| `winget` не найден | Обнови «App Installer» из Microsoft Store |
| `java`/`adb` не найдены после установки | Закрой ВСЕ окна терминала и открой заново; если не помогло — перезагрузка |
| `sdkmanager` не найден | Проверь, что переименовал папку в `latest` (шаг 3.1) и прописал Path (шаг 3.2) |
| Docker «WSL 2 not installed» | Выполни `wsl --install`, перезагрузись |
| Скачивание Android-инструментов оборвалось | Повтори блок 3.1 |
